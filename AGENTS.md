## Learned User Preferences

- When the user asks to play, test, or check out the game, start the dev server in the background and open the URL in the Cursor embedded browser via the `cursor-ide-browser` MCP rather than just printing a link.
- Switch into plan mode before any multi-step or irreversible change (folder restructures, branch deletions, new features, multi-minigame design); the user frequently says "go into plan mode" and expects a `CreatePlan` artifact written under `.cursor/plans/`.
- Commit directly to `main` for routine refactors and feature work in this repo; do not open PRs unless asked, and keep `.cursor/hooks/state/continual-learning.json` out of unrelated commits.
- The user often delegates execution of approved plans to the Composer 2 model, so plans should include a sharpened work-unit table with batchable vs sequential units and explicit verification commands per unit. Run `/simplify` and commit to main after feature implementation blocks.
- When executing an approved plan, do not edit the plan artifact or recreate preexisting to-dos; mark the provided to-dos in progress/completed as work proceeds and keep going until they are all resolved.
- Push back with ranked, opinionated creative options and a recommendation rather than hedging; the user prefers ambitious bets framed against deadlines and constraints over neutral lists.
- Bug-detective direction the user reinforces: keep the loop desk-led and exploratory (not only a speed trial for clues); the runner should take over full-screen arcade-style, use Mario-like run/jump on code-snippet platforms as the real floor (avoid a separate solid ground band), allow snippets to fade or expire so climbing stays urgent, keep scroll speed and gaps human-feasible, and steer visuals toward Cursor AI branding (reference official Cursor cosmetic and theme when tuning art); endless mode should ramp difficulty sharply and tie to leaderboards; the mascot should fall with believable gravity when unsupported (not hover in place on gaps); on the desk, mascot motion should feel like a small character (not floaty) and clue beats should read as a deliberate examine/detail zoom, not only framing props in the camera). In the runner, bind **jump to Space** (not Tab) as the primary key unless a documented conflict with focus or platform conventions forces otherwise.
- Bug Detective minigames should stay replayable after failure or success, with explicit restart paths and clear success conditions so puzzle clues feel earned rather than handed out after confusing endings.
- Dispatch/errand direction: favor a neon Cursor-terminal PvZ-style lane-defense loop with 3 horizontal lanes, agent selection via 1-4 plus lane deployment via Q/W/E or click, bug/cyber attackers that ramp into bosses, a clue threshold before endless scoring, and a friendly tutorial; keep it fun, visually distinct, and readable rather than a complex PvZ economy.
- For Bug Detective ideation, prioritize creative Cursor-themed mechanics that are accessible to non-technical adults; make scoring, goals, and end states obvious before implementation plans.
- Opening and onboarding: make the title or intro beat readable and pause long enough to read; animate the case/instruction sheet smoothly onto the desk and let it persist for rereading instead of flying vertically near the monitor and disappearing; on mascot reveal, prefer a jump-in and walk rather than emerging from the coffee or starting half-buried in the table; include high-level guidance for first-time players before free exploration.

## Learned Workspace Facts

- Repo is a flat monorepo at github.com/casper7995/cursor-crew with three self-contained app folders at the root: `shooting-game/` (Canvas jam game), `tower-defense/` (Three.js Defrag Run), and `bug-detective/` (active Vibe Jam 2026 build). For playtests when the user does not name a specific app, use `bug-detective/` rather than assuming another root app.
- Each app owns its own `package.json`, configs, and dev/test/build scripts; always `cd` into the app folder before running commands. `bug-detective/` is the active development target: Three.js + Vite + Vitest with a Cloudflare Worker under `bug-detective/worker/` that accepts a `puzzleId` to serve separate leaderboards (daily vs runner).
- The Vite dev server in this workspace usually binds to `http://127.0.0.1:5175/` because ports 5173 and 5174 are typically already in use.
- The Bug Detective **QA cockpit** (cloud agent / artifacts / Gemini loop UI) is **not** Vite: run `npm run qa:cockpit` from `bug-detective/` and open **`http://127.0.0.1:5875/`** (override with `QA_COCKPIT_PORT`). Use that single server for the cockpit; `npm run dev` is only the 3D game. If the page fails to load (`ERR_CONNECTION_REFUSED`) or the default port is already in use, start the cockpit or choose a free port with `QA_COCKPIT_PORT`.
- `docs/mcp-cursor-cloud.md` records the current MCP guidance: Cursor Cloud Agents MCP toggles are separate from local `~/.cursor/mcp.json`; Supabase hosted MCP can use PAT `Authorization` headers on clients that support custom headers; Vercel's hosted MCP at `https://mcp.vercel.com` is OAuth-based, and "app redirect URL is invalid" points to a Cursor/Vercel redirect allowlist issue rather than repo code.
- `bug-detective/qa-loop/` cockpit artifact sync is slot-aware: playable cloud artifacts are classified high/low confidence, ambiguous `bug-detective-*` clips require a saved `cockpit.artifactSlotOverrides[path]` assignment before download, Step 2 persists overrides through `/artifacts/assign`, and Gemini assessment runs once per minigame slot present in `manifest.videos` rather than choosing only the newest clip.
- `bug-detective/src/main.ts` is the orchestration choke point and the highest merge-conflict risk; isolate parallel work into `bug-detective/src/minigames/` or other new modules and serialize edits to `main.ts` and `src/game/gameState.ts`.
- Cursor's embedded browser often mishandles input compared to desktop Chrome: avoid relying on `pointerdown` on `window` alone for global UI—use **document capture-phase** listeners (`pointerdown`, `mousedown`, `click`, `touchstart`) for title splash dismiss, keep dismiss layers above widgets like the Vibe Jam badge with a very high `z-index`, and drive desk gameplay from **pointer** events on the canvas and HUD (`pointermove` / `pointerdown`, not only `mousemove`) or the intro peel and investigation hover raycasts stall when the user clicks without moving the mouse.
- Bug Detective zoom interactions have recurring trap risk: every focus/prop zoom needs an obvious zoom-out affordance, `Esc` support, bounded camera distance, and auto-unzoom before navigating to another location. Prefer a small number of clear zoom levels (e.g. in and out) over many free angles so users are less likely to get stuck between states.
- The mascot must read as Cursor AI style: transparent tilted cube head with a centered inverted downward-pointing inner pyramid/very dark black cursor wedge (top/left/right faces meeting at one bottom point, right triangle a perfect 50% cut), fluid round half-black/half-white or glass body, wider-set vertical oval eyes, broad smile matched to `bug-detective/Cursor-Mascot-Reference.jpeg`, and consistent 3D desk plus 2D runner silhouettes.
- Desk pointer rays hit the desk plane under tall props, so feet targets would otherwise sit inside mug/monitor volumes; `bug-detective/src/cursor/deskFootResolve.ts` pushes feet out of `mascotFootObstacles` from `desktopDiorama.ts`, wired via `cursorTracker` in `main.ts` (see `bug-detective/tests/deskFootResolve.test.ts`).
- `bug-detective/src/minigames/errand/` is the Cursor Agents lane-defense notebook minigame: 3 horizontal lanes, agents such as Fixer / Reviewer / Firewall / Debugger selected with 1-4 and deployed with Q/W/E or lane clicks, neon Cursor-terminal PvZ-style board, Focus/CAP/BASE meters, wave escalation, Zero-Day bosses, and clue lock once wave 3 or 60 seconds is reached before endless score chasing.
